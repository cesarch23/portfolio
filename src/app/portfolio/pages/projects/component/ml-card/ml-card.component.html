Modelo predictivo de tipo de diabetes

Objetivo: Desarrollar un modelo de Machine Learning para clasificar el tipo de diabetes (Tipo mellitis tipo 1 vs. mellitipos tipo 2) basándose en datos demográficos y de historial médico de pacientes.

Descripción del Proyecto:
Este proyecto abarca un ciclo completo de ciencia de datos, desde la ingesta y limpieza de datos hasta la optimización y evaluación de modelos predictivos.

1. Carga y Preprocesamiento de Datos:

  • Se consolidaron múltiples archivos CSV de afiliados activos con diagnóstico de diabetes (más de 5 millones de registros iniciales).
  • Se realizó una selección y renombrado de columnas, eliminando información redundante o irrelevante.
  • Se filtraron los datos para incluir solo los tipos de diabetes relevantes (Tipo 1 y Tipo 2), resultando en un dataset de 57,603 registros únicos.

2. Calidad y Limpieza de Datos:

  • Se optimizaron los tipos de datos para mejorar la eficiencia de memoria (ej., int8, category).
  • Se identificaron y eliminaron registros duplicados.
  • Se verificó la ausencia de valores nulos en el dataset.

3. Detección y Tratamiento de Outliers:

  • Se aplicaron técnicas basadas en el Rango Intercuartílico (IQR) para identificar y corregir valores atípicos en variables numéricas clave como edad, cant_atenciones y dias_hosp, utilizando un método de capping para mantener la integridad de los datos.

4. Análisis Exploratorio de Datos (EDA):

  • Se llevó a cabo un análisis univariado para comprender la distribución de variables categóricas y numéricas mediante gráficos de barras, histogramas y estadísticas descriptivas.
  • Se exploraron las relaciones bivariadas entre variables utilizando boxplots, scatter plots y un mapa de calor para identificar correlaciones entre las características numéricas.

5. Transformación de Características para Modelado:

  • Se verificó la ausencia de multicolinealidad entre las variables predictoras numéricas.
  • El dataset se dividió en conjuntos de entrenamiento, validación y prueba, manteniendo la proporción de las clases objetivo mediante la estratificación.
  • Se aplicó One-Hot Encoding a las variables categóricas (sexo, con_dx_obesidad, con_dx_saludmental).
  • Se utilizó Robust Scaling para normalizar las variables numéricas, haciéndolas robustas a outliers.
  • La variable objetivo (tipo_diabetes) fue codificada numéricamente para el entrenamiento del modelo.

6. Desarrollo y Optimización del Modelo:

  • Se construyó un modelo de clasificación inicial utilizando RandomForestClassifier.
  • Se evaluó la estabilidad del modelo mediante validación cruzada (cross-validation) y se utilizaron métricas como f1-score.
  • Se implementó una búsqueda aleatoria (RandomizedSearchCV) con validación cruzada estratificada (StratifiedKFold) para optimizar los hiperparámetros del RandomForestClassifier, enfocándose en maximizar la métrica recall para la clase minoritaria.

7. Evaluación del Modelo:

  • Se evaluó el rendimiento del modelo optimizado en los conjuntos de entrenamiento y validación utilizando matrices de confusión y reportes de clasificación, observando mejoras significativas en las métricas clave post-optimización.

Tecnologías/Librerías: Python, Pandas, NumPy, Matplotlib, Seaborn, Scikit-learn (RandomForestClassifier, OneHotEncoder, RobustScaler, train_test_split, RandomizedSearchCV, StratifiedKFold, classification_report, confusion_matrix, cross_val_score).
